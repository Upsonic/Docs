## Overview

Server-Sent Events (SSE) based servers for remote MCP tool access.

## Usage

```python
from upsonic import Task, Agent
from upsonic.tools.mcp import MCPHandler

# SSE-based MCP servers
fetch_handler = MCPHandler(
    url="https://your-mcp-server.com/sse",
    transport="sse",
    timeout_seconds=60
)

search_handler = MCPHandler(
    url="https://search-mcp-server.com/sse",
    transport="sse",
    timeout_seconds=60
)

# Create task with MCP tools
task = Task(
    description="Search for the latest technology news in Europe using the search tool, then retrieve the full content of the top 3 articles.",
    tools=[fetch_handler, search_handler]
)

# Create agent
agent = Agent(
    name="News Agent",
    model="anthropic/claude-sonnet-4-5"
)

# Execute
result = agent.print_do(task)
print("Result:", result)
```

## Parameters

- `url` (str): The SSE endpoint URL of the MCP server
- `transport` (str): Must be `"sse"` when using `url`
- `timeout_seconds` (int): Connection timeout in seconds (default: 5)
- `tool_name_prefix` (str): Optional prefix to add to all tool names (prevents collisions)

## Using Tool Name Prefix

Add a `tool_name_prefix` when using multiple SSE-based MCP servers with overlapping tool names:

```python
from upsonic.tools.mcp import MCPHandler

fetch_handler = MCPHandler(
    url="https://fetch-mcp.example.com/sse",
    transport="sse",
    tool_name_prefix="fetch",
    timeout_seconds=60
)

search_handler = MCPHandler(
    url="https://search-mcp.example.com/sse",
    transport="sse",
    tool_name_prefix="search",
    timeout_seconds=60
)
```

## Characteristics

- Remote server access over HTTP
- Real-time event streaming
- Suitable for cloud-hosted MCP servers
- No local installation required
