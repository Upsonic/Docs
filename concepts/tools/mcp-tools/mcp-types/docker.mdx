## Overview

Docker container-based MCP servers for maximum isolation.

## Usage

```python
from upsonic import Task, Agent
from upsonic.tools.mcp import MCPHandler

# Docker-based MCP server
docker_handler = MCPHandler(
    command="docker run --rm -i --network host your-mcp-server-image:latest",
    env={
        "DATABASE_URL": "postgresql://localhost/mydb"
    },
    timeout_seconds=60
)

# Create task
task = Task(
    description="Connect to the PostgreSQL database container, query the latest 100 transactions, and analyze them for potential anomalies.",
    tools=[docker_handler]
)

# Create agent
agent = Agent(
    name="Database Agent",
    model="anthropic/claude-sonnet-4-5"
)

# Execute
result = agent.print_do(task)
print("Result:", result)
```

## Parameters

- `command` (str): The command string including "docker run" and image name with arguments
- `env` (Dict[str, str]): Environment variables to pass to container
- `timeout_seconds` (int): Connection timeout in seconds (default: 5)
- `tool_name_prefix` (str): Optional prefix to add to all tool names (prevents collisions)

## Using Tool Name Prefix

Add a `tool_name_prefix` when using multiple Docker-based MCP servers:

```python
from upsonic.tools.mcp import MCPHandler

prod_handler = MCPHandler(
    command="docker run --rm -i postgres-mcp:latest",
    env={"DATABASE_URL": "postgresql://prod-server/db"},
    tool_name_prefix="prod_db",
    timeout_seconds=60
)

staging_handler = MCPHandler(
    command="docker run --rm -i postgres-mcp:latest",
    env={"DATABASE_URL": "postgresql://staging-server/db"},
    tool_name_prefix="staging_db",
    timeout_seconds=60
)
```

## Requirements

- Docker must be installed and running
- MCP server Docker image must be available

## Characteristics

- Maximum isolation and security
- Consistent environment across systems
- Easy deployment and versioning
- Network and resource control
